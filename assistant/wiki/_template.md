# AI Wiki 공통 템플릿

`data/` 폴더의 원본 문서(사용자 매뉴얼, 용어 사전, 메뉴 사용법 등)를 AI Wiki 형태로 변환할 때
공통으로 사용하는 템플릿이다. RAG 검색 시 Vector 검색과 Metadata Filter를 함께 사용할 수 있도록
문서 상단에 YAML Front Matter로 메타데이터를 붙이고, 본문은 아래의 공통 섹션 구조를 따른다.

---

```markdown
---
id:                 # 문서 고유 ID (예: MANUAL-021)
title:              # 문서 제목
doc_type:           # 사용자 매뉴얼 | 용어 사전 | 메뉴 가이드 | 기타
category:           # 소속 시스템/플랫폼명
version:            # 문서 버전 (예: v1.0)
created_date:       # 원본 작성일 (YYYY-MM-DD)
updated_date:       # 최종 수정일 (YYYY-MM-DD)
approval_status:    # pending | approved | rejected
approved_by:        # 승인자 (미승인 시 null)
tags: []            # 검색/필터용 키워드 목록
source_file:        # 원본 파일 경로 (data/...)
related_menus: []   # 연관 메뉴명
summary:            # 1~2문장 요약 (임베딩 시 대표 텍스트로 활용 가능)
---

# {{title}}

## 개요
문서가 다루는 대상과 목적을 요약한다.

## 예상 질문
사용자가 이 문서를 근거로 질문할 만한 자연어 질문 예시를 나열한다.
(AI Wiki Retriever의 검색 품질 검증 및 테스트 질의로 활용)

## 주요 내용
원본 문서의 핵심 내용을 기능/메뉴/주제 단위로 나눈 하위 섹션(###)으로 정리한다.
표, 절차, 코드블록 등 원본 형식은 최대한 유지한다.

## 관련 용어
문서에서 등장하는 핵심 용어를 표(용어 | 설명)로 정리한다. (해당 없으면 섹션 생략)

## 관련 메뉴 / 업무 절차
문서와 연결되는 메뉴, 다음 단계, 업무 흐름을 정리한다. (해당 없으면 섹션 생략)

## FAQ
자주 묻는 질문과 답변을 정리한다. (해당 없으면 섹션 생략)

## 참고 자료 / 출처
원본 문서명과 경로를 명시한다.
```

---

## 섹션 사용 규칙

| 섹션 | 필수 여부 | 비고 |
| --- | --- | --- |
| Front Matter(메타데이터) | 필수 | 모든 문서 공통 |
| 개요 | 필수 | 1~3문장 |
| 예상 질문 | 권장 | 검색 테스트 질의로 재사용 |
| 주요 내용 | 필수 | 원본 섹션을 ### 단위로 분해 |
| 관련 용어 | 선택 | 용어 사전형 문서는 주요 내용에 통합 가능 |
| 관련 메뉴 / 업무 절차 | 선택 | 메뉴 가이드형 문서에 필수 |
| FAQ | 선택 | 원본에 FAQ가 있는 경우만 |
| 참고 자료 / 출처 | 필수 | 원본 추적성 확보 |

## 메타데이터 필드 정의

| 필드 | 설명 | 예시 |
| --- | --- | --- |
| id | 문서 고유 식별자 | MANUAL-021 |
| doc_type | 문서 유형 | 사용자 매뉴얼 / 용어 사전 / 메뉴 가이드 |
| category | 소속 시스템/플랫폼 | AI 비전 검사 시스템 |
| version | 문서 버전 | v1.0 |
| created_date / updated_date | 작성일 / 수정일 | 2026-07-02 |
| approval_status | 승인 상태 (Wiki는 검수·승인 후 답변 근거로 사용) | pending / approved / rejected |
| tags | 검색 및 Metadata Filter용 키워드 | [비전검사, GOOD/DEFECT] |
| source_file | 원본 파일 경로 | data/21 시스템 사용자 매뉴얼.md |
| related_menus | 연관 메뉴 | [리뷰, 모델관리] |
