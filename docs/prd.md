# PRD — Knowledge Space 생성

## 1. 배경 및 목적
사내/팀에 흩어져 있는 문서(기획서, 스펙, DB 스키마 등)를 업로드하면 AI가 자동으로 위키 형태로 구조화하고, 사람이 검토·승인한 문서만 신뢰 가능한 지식베이스(Knowledge Space)로 축적한다. 축적된 위키는 자연어 질의응답(Q&A)으로 탐색할 수 있다.

## 2. 문제 정의
- 문서가 여러 파일/포맷(PDF, DOCX, TXT, MD, SQLite)에 흩어져 있어 검색·활용이 어렵다.
- AI가 자동 정리한 내용은 오류나 누락 가능성이 있어 그대로 신뢰하기 어렵다.
- 팀/주제별로 지식을 격리해서 관리할 필요가 있다.

## 3. 목표
- 파일 업로드 → AI 분석 → 사람 검토(승인/반려) → 위키 축적까지 하나의 파이프라인으로 제공한다.
- 승인된 문서에 한해 자연어 질의응답을 지원한다.
- 여러 개의 독립된 Space(주제/팀 단위)를 생성하고 전환할 수 있다.

## 4. 비목표 (Out of scope)
- 실제 파일 저장/파싱 백엔드 연동 (본 문서는 프로토타입 UI 기준이며, 실제 구현 범위는 api-spec.md 참조)
- 문서 공동 편집, 코멘트 스레드
- 권한/조직 관리(팀 초대, 역할 분리) — 데모 수준의 사용자 전환만 지원
- 다국어 UI (한국어 전용)

## 5. 사용자
- **Space 소유자/멤버**: 문서를 업로드하고 검토·승인하는 실무자
- **탐색자**: 승인된 위키를 자연어로 질의하는 사용자
(데모에서는 한 화면에서 사용자를 전환하며 두 역할을 모두 수행)

## 6. 핵심 개념
| 개념 | 설명 |
|---|---|
| Knowledge Space | 이름/설명을 가진 독립된 지식 공간. 고유 `space_id` 보유. 이하 본 문서에서는 Space로 줄여 부른다 |
| 문서 등록(업로드) | Space에 원본 파일을 추가하는 행위 |
| 분석(Analysis) | 업로드된 파일을 AI가 처리해 1개 이상의 구조화 문서(요약/키포인트/태그/표 등)로 변환하는 비동기 과정 |
| 검토(Review) | 분석된 문서를 사람이 읽고 승인/반려하는 과정 |
| 위키(Wiki) | Space 내 승인된 문서들의 모음. 버전과 이력을 가짐 |
| 질문하기(Chat) | 위키(승인된 문서)에 근거해 자연어로 답하는 Q&A |

## 7. 기능 요구사항

### 7.1 Space 관리
- Space가 하나도 없으면 온보딩 화면(빈 상태)을 보여주고 생성 유도.
- Space 생성 모달: 이름(필수), 한 줄 설명(선택) 입력 → 생성 시 `space_id` 발급, 성공 화면 표시.
- 생성 완료 후 "문서 등록하러 가기"로 바로 이동.
- 좌측 사이드바에서 Space 목록 확인/전환, 접기/펼치기 지원.
- Space는 이름/문서 수를 사이드바에 표시.

### 7.2 문서 등록 (업로드)
- 드래그앤드롭 또는 파일 선택으로 다중 업로드.
- 지원 포맷: PDF, DOCX, TXT, Markdown, SQLite(db/sqlite/sqlite3). 미지원 포맷은 업로드 실패로 표시.
- 업로드된 파일은 대기(idle) 상태로 리스트에 표시되며, "분석시작" 버튼으로 분석 개시.
- 분석 중에는 단계별 진행 메시지를 순차 표시(스피너 + 메시지).
- 분석 중 실패 가능(랜덤) → 실패 표시 + "다시 시도".
- 분석 완료 시 파일 하나에서 1개 이상의 문서(SQLite는 테이블별 다수 문서)가 생성됨.
- 각 생성 문서는 검토 대기(pending) 상태로 시작하며, 이상 징후가 있으면 플래그(⚠) 개수 표시.
- 파일 항목 삭제 가능(진행 중 분석은 취소됨).

### 7.3 문서 검토
- 문서 상세 화면에서: 제목, 원본 파일명, 상태 배지, 확인 필요 플래그 목록, 섹션별 내용(텍스트/태그/표), 관련 문서, 버전 이력을 확인.
- Pending 문서: 승인 또는 반려 가능. 반려 시 사유 입력 필수(미입력 시 "사유 미입력"으로 기록).
- 승인 시: 버전 +1, 이력에 기록, 위키 화면으로 이동.
- 반려된 문서: 반려 사유 표시, "다시 검토 대기로" 전환 가능.
- 관련 문서 클릭 시 해당 문서 검토 화면으로 이동.

### 7.4 위키
- Space 내 승인된 문서를 카드 목록으로 표시(제목, 원본 파일명, 요약, 태그, 버전).
- 승인된 문서가 없으면 안내 문구 + 문서 등록 화면 이동 버튼.
- 카드 클릭 시 검토(상세) 화면으로 이동(읽기 전용처럼 동작, 승인 완료 배지 표시).

### 7.5 질문하기 (Q&A)
- 승인된 문서가 없으면 질의 불가 안내 표시.
- 자연어 질문 입력 → 승인된 문서 전체를 컨텍스트로 하여 답변 생성.
- 답변에는 근거가 된 문서 제목을 출처 칩으로 함께 표시.
- 답변 생성 중 "위키를 참고해서 답변을 정리하고 있어요…" 로딩 상태 표시.

### 7.6 공통
- 다크/라이트 테마 토글.
- 데모용 사용자 전환(계정 스위처).
- "전체 초기화" 버튼으로 모든 데이터 리셋(데모 편의 기능).

## 8. 문서 상태 흐름
```
idle(업로드됨) → analyzing(분석중) → done(분석완료, 문서 N개 생성)
                                   → analysis_failed(재시도 가능)
파일 미지원 포맷 → upload_failed

문서: pending → approved (버전 +1, 위키 등록)
              → rejected (사유 기록) → pending(재검토 요청) 가능
```

## 9. 성공 기준 (데모/과제 기준)
- 빈 상태 → Space 생성 → 파일 업로드 → 분석 → 검토 → 승인 → 위키 노출 → 질문하기까지 끊김 없이 시연 가능.
- 여러 Space를 만들고 전환해도 각 Space의 데이터(파일/문서/채팅)가 독립적으로 유지됨.
- 승인되지 않은 문서는 Q&A 컨텍스트에 포함되지 않음.
