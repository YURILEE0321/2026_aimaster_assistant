# 디자인 명세 — Knowledge Space 생성

기준 파일: `Knowledge Space 생성.dc.html` (동작하는 프로토타입 기준으로 작성됨)

## 0. 디자인 토큰

### 색상 (다크 / 라이트)
| 토큰 | 다크 | 라이트 | 용도 |
|---|---|---|---|
| `--bg` | `#0B0E13` | `#F6F3EC` | 배경 |
| `--text` | `#F6F3EC` | `#181B14` | 본문 텍스트 |
| `--accent-text` | `#FF8A3D` | `#B34A12` | 강조색(브랜드 오렌지) |
| ink rgb (텍스트 계열 투명도용) | `246,243,236` | `24,27,20` | `rgba(var(--ink-rgb), α)` |

보조 색상: 위험/반려 `#B25A3E` / `#E08A6C`(라이트 텍스트), 모달 표면 `#F6F3EC`(카드/모달은 라이트 배경 고정), 모달 본문 텍스트 `#181B14`.

### 타이포그래피
- 제목/헤드라인: `Newsreader` (serif, weight 500, 이탤릭 500 보조) — 감성적/에디토리얼 톤
- 본문/UI: `IBM Plex Sans KR` (400/500/600)
- 코드/라벨/메타정보: `IBM Plex Mono` (400/500) — space_id, 파일 크기, 타임스탬프, 테이블 헤더 등

### 스케일 (참고)
- H1(온보딩): 52px / H1(Space 상세): 36px / H1(검토): 32px
- 모달 타이틀: 24–26px / 섹션 소제목: 14.5–15px / 본문: 13–15px / 캡션·메타: 10.5–12px

### 간격/형태
- 코너 반경: 카드/입력 8–16px, 배지/태그는 pill(16–20px)
- 카드/패널 배경: `rgba(ink,0.02~0.03)` + `rgba(ink,0.06~0.14)` 보더
- 강조 버튼: 오렌지 배경(`#FF8A3D`) + 다크 텍스트(`#0B0E13`)
- 그림자: 모달 `0 30px 80px rgba(0,0,0,0.5)`, 유저메뉴 `0 16px 40px rgba(0,0,0,0.45)`

### 모션
- 진입: `fadeUp`(0.3–0.7s), 모달/드롭다운: `scaleIn`(0.18–0.32s), 배경 dim: `dimIn`(0.25s)
- 로딩 스피너: `spin` 0.8s linear infinite
- CTA 유도(빈 상태 버튼): `pulseRing` 2.4s 반복
- 배경 장식(블롭): `drift` 18–22s ease-in-out infinite

## 1. 전역 레이아웃
- 최상단 헤더(고정 아님, flex-shrink:0): 좌측 로고("W" 마크 + "Wikigen"), 우측 테마 토글 + 계정 메뉴.
- 계정 메뉴: 아바타(이니셜) + 이름/이메일 + 드롭다운 화살표. 클릭 시 유저 목록 드롭다운(라이트 표면 고정), 배경 클릭 시 닫힘.
- 배경: 은은한 도트 텍스처 + 좌우 블롭 그라디언트(오렌지/블루), `pointer-events:none`.

## 2. 화면 A — 빈 상태 (온보딩)
- 조건: Space 0개.
- 중앙 정렬, 세로 스택: 오버라인("KNOWLEDGE SPACE") → H1("아직 만들어진 space가 없습니다") → 설명 문단 → CTA 버튼("＋ 새 Space 만들기").
- CTA는 pulseRing 애니메이션으로 지속적 주목 유도.

## 3. 화면 B — Space 목록/상세 (메인)
### 3.1 사이드바 (좌, 264px / 접힘 64px)
- 펼침: 헤더("Spaces · N", ＋ 생성, ◀ 접기) + Space 항목 리스트(이름, 문서 수, 활성 시 좌측 액센트 바 + 배경 tint).
- 접힘: 아이콘만(▶ 펼치기, ＋, Space 이니셜 원형 버튼).

### 3.2 메인 콘텐츠
- 상단: 활성 Space의 `SPACE · {space_id}` 라벨(오렌지 도트) → Space명(H1, Newsreader) → 설명.
- 탭 3개(밑줄 인디케이터, 활성 시 오렌지): 문서 등록 / 위키 · N / 질문하기.

#### 3.2.1 탭 — 문서 등록
- 헤더 우측에 "N개 파일 등록됨" 카운터.
- 드롭존: 점선 보더, 업로드 아이콘, 안내 문구("파일을 드래그하거나 클릭하여 업로드"), 지원 포맷 캡션, "파일 선택" 버튼.
- 업로드된 파일 카드(리스트, gap 8px): 파일 아이콘 + 이름 + 크기 · 우측 상태 액션(분석시작 / 분석완료 배지 / 실패+다시시도 / 업로드실패) · 삭제(×).
  - 분석 중: 하위 행에 스피너 + 진행 메시지.
  - 분석 실패/업로드 실패: 경고 배너(⚠ + 안내문).
  - 완료 시 하위에 생성된 문서 목록(좌측 보더 인덴트): 문서명 + 플래그 카운트 + 상태 배지 + 액션 버튼(검토하기/다시 보기/다시 검토).

#### 3.2.2 탭 — 위키 · N
- 승인 문서 있음: 카드 리스트(제목 + 📘 아이콘 + 버전 라벨, 원본 파일명(모노), 설명 1~2줄, 태그 pill 목록). 클릭 시 검토 화면 이동.
- 없음: 점선 박스 안내 문구 + "문서 등록으로 이동" 버튼.

#### 3.2.3 탭 — 질문하기
- 승인 문서 없을 때: 상단 경고 배너.
- 채팅 패널(고정 높이 520px, 라운드 보더): 메시지 리스트(유저=우측 오렌지 버블 / 어시스턴트=좌측 뉴트럴 버블) + 출처 칩(📘 문서명) + 로딩 인디케이터(스피너+문구) → 하단 입력창(Enter 전송) + 전송 버튼.
- 빈 상태: 중앙 안내 문구.

- 화면 최하단: "전체 초기화 (데모)" 텍스트 링크(우측 정렬, 저채도).

## 4. 화면 C — 문서 검토 (Review)
- 상단: "← 목록으로" + 원본 파일명(브레드크럼 스타일).
- 타이틀 영역: 문서 제목(H1) + 버전 라벨 + 상태 배지(검토대기/승인됨/반려됨, 색상은 `statusStyle` 매핑).
- 플래그 배너(있을 때만): "확인이 필요해요" + 항목별 ⚠ 리스트.
- 섹션 반복 렌더(heading은 오렌지 소제목):
  - text 섹션: 문단 스택
  - tags 섹션: pill 태그 wrap
  - table 섹션: 얇은 보더 테이블, 헤더는 모노스페이스 대문자
- "관련 문서" 섹션: 버튼 칩 목록(클릭 시 해당 문서로 재진입) 또는 "관련 문서가 없어요".
- 반려된 경우: 반려 사유 배너.
- "버전 이력" 섹션: 점(dot) + 라벨 + 타임스탬프 타임라인(최신순).
- 하단 스티키 액션바(블러 배경):
  - pending: [반려] [승인] — 반려 클릭 시 사유 입력 인풋 + [취소]/[반려 확정]으로 전환.
  - approved: "✓ 승인 완료" 라벨 + [목록으로].
  - rejected: [다시 검토 대기로] [목록으로].

## 5. 모달 — Space 생성
- 배경 dim + blur, 카드는 라이트 표면 고정(다크 모드에서도 동일 — 대비를 위한 의도적 선택).
- Form 상태: 오버라인("새 KNOWLEDGE SPACE") → 타이틀("Space 만들기") → 이름 입력(필수, 에러 시 빨간 보더+메시지) → 설명 입력(선택) → [취소]/[Space 생성].
- Success 상태: 체크 아이콘(다크 원형) → "{name} 생성 완료" → `space_id` 모노스페이스 배지 → "문서 등록하러 가기 →" 풀와이드 버튼.

## 6. 상태 배지 색상 규칙
| 상태 | 배경 | 텍스트 |
|---|---|---|
| 검토 대기(pending) | `rgba(ink,0.08)` | `rgba(ink,0.6)` |
| 승인됨(approved) | `rgba(255,138,61,0.14)` | accent-text |
| 반려됨(rejected) | `rgba(178,90,62,0.16)` | `#E08A6C` |

## 7. 반응형/접근성 메모
- 현재 프로토타입은 데스크톱 고정 레이아웃(1280px+ 가정) 기준. 모바일 대응은 범위 밖(과제 범위에서 필요 시 별도 논의).
- 인터랙티브 요소 hit target은 24–32px 아이콘 버튼 다수 — 실제 제품화 시 44px 이상 확대 검토 필요.
- 포커스 상태: 입력창 `style-focus`로 보더 컬러 강조(오렌지/레드) — 키보드 포커스 링 별도 미정의, 접근성 개선 시 추가 검토.
