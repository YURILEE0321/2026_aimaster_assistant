# 사용자 시나리오 — Knowledge Space 생성

## 시나리오 1. 첫 Space 만들기 (온보딩)
1. 홍길동이 서비스에 처음 진입한다. Space가 없어 "아직 만들어진 space가 없습니다" 안내 화면을 본다.
2. "새 Space 만들기" 버튼을 누른다.
3. 모달에서 이름 "백엔드 아키텍처 문서", 설명 "백엔드 팀 기술 문서 모음"을 입력하고 생성한다.
4. `space_id`가 발급되고 생성 완료 화면이 뜬다.
5. "문서 등록하러 가기"를 눌러 바로 업로드 화면으로 이동한다.

**대안 흐름**: 이름을 비운 채 생성 시도 → 인라인 에러 메시지 표시, 생성되지 않음.

## 시나리오 2. 문서 업로드 및 분석
1. 홍길동이 `api-design.pdf`, `schema.sqlite` 두 파일을 드래그앤드롭으로 업로드한다.
2. 두 파일이 리스트에 대기 상태로 추가된다.
3. 각 파일의 "분석시작"을 누른다.
4. 분석 중 진행 메시지("문서를 확인하고 있어요" → … → "마무리하는 중이에요")가 순차적으로 바뀐다.
5. `api-design.pdf`는 문서 1개로, `schema.sqlite`는 테이블별 문서(users/orders/products) 3개로 분석 완료된다.
6. 일부 문서에는 "확인이 필요해요" 플래그가 붙어 있다.

**대안 흐름 (분석 실패)**: 분석 도중 실패 표시가 뜨면 "다시 시도"를 눌러 재분석한다.
**대안 흐름 (미지원 포맷)**: `.zip` 파일 업로드 시 즉시 "업로드 실패 · 지원하지 않는 파일 형식" 표시.

## 시나리오 3. 문서 검토 및 승인
1. 홍길동이 `users 테이블` 문서의 "검토하기"를 누른다.
2. 검토 화면에서 요약, 핵심 포인트, 키워드 태그, 컬럼 구조 표, 관련 문서(orders/products), 버전 이력을 확인한다.
3. 플래그가 있다면 내용을 확인 후 문제 없다고 판단해 "승인"을 누른다.
3.1. 승인 버튼을 누르면 위키 내용에 대한 청크/임베딩 작업을 수행한다.
4. 문서가 v1으로 승인되고 자동으로 위키 화면으로 이동, 카드로 노출된다.

**대안 흐름 (반려)**: 내용에 오류가 있다고 판단해 "반려"를 누르고 사유("컬럼 설명이 부정확함")를 입력 후 "반려 확정". 문서는 반려 상태가 되며, 이후 "다시 검토 대기로"를 눌러 재검토 큐에 되돌릴 수 있다.

## 시나리오 4. 위키 열람
1. 이유리가 사이드바에서 "백엔드 아키텍처 문서" Space를 선택한다.
2. "위키" 탭으로 이동해 현재까지 승인된 문서(users, orders 테이블) 카드를 확인한다.
3. 카드를 클릭해 상세 내용을 다시 확인한다(승인 완료 배지 확인).

**대안 흐름**: 아직 승인된 문서가 없는 Space는 안내 문구와 "문서 등록으로 이동" 버튼만 보인다.

## 시나리오 5. 자연어 질문하기
1. 박정병이 "질문하기" 탭으로 이동한다.
2. "orders 테이블의 결제 금액 컬럼은 어떤 타입이야?"라고 입력하고 전송한다.
3. "위키를 참고해서 답변을 정리하고 있어요…" 로딩 후, 승인된 문서(orders 테이블)를 근거로 한 답변과 출처 칩("📘 orders 테이블")이 표시된다.

**대안 흐름 (승인 문서 없음)**: 승인된 문서가 하나도 없는 Space에서는 안내 배너가 뜨고, 질문을 보내도 "승인된 문서가 없어서 답변할 수 없다"는 고정 응답을 받는다.

## 시나리오 6. 여러 Space 간 전환
1. 이기훈이 "프론트엔드 컨벤션"이라는 두 번째 Space를 새로 만든다.
2. 사이드바에서 두 Space("백엔드 아키텍처 문서", "프론트엔드 컨벤션")를 오가며 각각의 문서 목록/위키/채팅이 서로 섞이지 않고 독립적으로 유지됨을 확인한다.
3. 사이드바를 접어 아이콘 모드로 축소한 뒤에도 Space 전환이 가능하다.

## 시나리오 7. 사용자/테마 전환 (데모 편의)
1. 우측 상단 계정 메뉴에서 사용자를 "홍길동"에서 "이유리"로 전환한다.
2. 다크 모드 아이콘을 눌러 라이트 테마로 전환해 가독성을 비교한다.
3. 데모를 마친 뒤 "전체 초기화"를 눌러 모든 Space/문서/대화를 지운다.
